In this guide, we'll talk about how you can integrate Web3Auth Plug and Play(PnP) Web SDKs with
Scaffold ETH, to leverage Web3Auth's social authentication processes alongside RainbowKit's
user-friendly wallet management interface.

As an overview, we'll use
[Web3Auth Wagmi Connector](https://web3auth.io/docs/sdk/pnp/web/wagmi-connector) to bridge Web3Auth
with RainbowKit, facilitating a secure and efficient authentication mechanism for users. For those
who want to skip straight to the code, you can find it on GitHub -
[Scaffold ETH Modal Example](https://github.com/Web3Auth/web3auth-pnp-examples/tree/main/web-modal-sdk/wagmi-examples/scaffold-eth-modal-example),
[Scaffold ETH No Modal Example](https://github.com/Web3Auth/web3auth-pnp-examples/tree/main/web-no-modal-sdk/wagmi-examples/scaffold-eth-no-modal-example).

## Crate a Scaffold-ETH 2 project

Once, you have set up the Web3Auth Dashboard, and created a new project, it's to create a
Scaffold-ETH 2 project. For this guide, we are going to use Scaffold-ETH 2. If you haven't created
the project already,
[please follow the steps on Scaffold-ETH documentation](https://scaffold-eth-2-docs.vercel.app/quick-start/installation).

## Integrating Web3Auth with Scaffold ETH

After creating a new project it's time to integrate Web3Auth PnP Web SDK with Scaffold-ETH. For the
implementation, we'll use PnP Modal SDK, and PnP No Modal SDK. The Modal SDK provides a prebuilt UI
modal interface, where as the No Modal SDK provides direct integration, by passing the modal for a
more customizable UI.

### Installation

Based on your Web PnP SDK preference, you need to install the following pacakges. Along with Web SDK
packages, you also need to install
[@web3auth/web3auth-wagmi-connector](https://www.npmjs.com/package/@web3auth/web3auth-wagmi-connector)
to create a Web3Auth connector.

```bash npm2yarn
$ npm i @web3auth/web3auth-wagmi-connector
```

### Create Connector for Modal SDK

Once you have install the relevant packages, let's look at creating a connector using Web3Auth Wagmi
Connector package.

We'll use the `scaffoldConfig` to get the targated networks, and use the first one to create
`chainConfig` for `EthereumPrivateKeyProvider`. Once you have created `Web3Auth` instance, you can
create a `Web3AuthConnector` for wagmi.

```tsx
const chain = scaffoldConfig.targetNetworks[0];

const chainConfig = {
  chainNamespace: CHAIN_NAMESPACES.EIP155,
  chainId: "0x" + chain.id.toString(16),
  rpcTarget: chain.rpcUrls.default.http[0],
  displayName: chain.name,
  tickerName: chain.nativeCurrency?.name,
  ticker: chain.nativeCurrency?.symbol,
  blockExplorerUrl: chain.blockExplorers?.default.url[0] as string,
};

// Additional code to create Web3Auth Modal instance.

export const rainbowWeb3AuthConnector = (): Wallet => ({
  id: "web3auth",
  name: "Web3Auth",
  rdns: "web3auth",
  iconUrl: "https://web3auth.io/images/web3authlog.png",
  iconBackground: "#fff",
  installed: true,
  downloadUrls: {},
  createConnector: (walletDetails: WalletDetailsParams) =>
    createWagmiConnector((config) => ({
      ...Web3AuthConnector({
        web3AuthModalInstance,
      })(config),
      ...walletDetails,
    })),
});
```

To checkout the whole code, please checkout this file. Once you have implemented
`rainbowWeb3AuthConnector`, you can simply update the wallets list inside `wagmicConnectors` file.

```tsx
const wallets = [
  // highlight-next-line
  rainbowWeb3AuthConnector,
  metaMaskWallet,
  walletConnectWallet,
  // Additional wallets
];
```
